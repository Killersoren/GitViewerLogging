# GitViewerLogging API

Logging service for Gitviewer. It consumes log messages generated by GitViewer.

Related repositories:

* **[GitViewer](https://github.com/Killersoren/GitViewer)** – Main backend API
* **[GitViewerClient](https://github.com/Killersoren/GitViewerClient)** – React frontend

## Description

GitViewerLogging is the service responsible for consuming log messages and storing them for later querying.

It exposes REST endpoints that allows the client to retrieve log data.

The service is built using **ASP.NET Core** and **Entity Framework Core**.

## Getting Started

### Requirements

* .NET 9 SDK
* Docker

### Running the program locally

1. Ensure **Docker is running**.
2. Start a **RabbitMQ container**.
3. Run the application:

```
dotnet run --project AppHost
```

The API will start locally and expose REST endpoints for repository and user operations.

A **PostgreSQL Docker container** will automatically be started for the local database.
Database data is stored in a persistent Docker volume.
